## 1. Purpose
This Password Policy defines the requirements for creating, managing
and protecting passwords used to access Novalink Technologies Ltd
information systems and data. Strong password practices are a
fundamental control that protects Novalink from unauthorized access
and data breaches.

---

## 2. Scope
This policy applies to:
- All Novalink employees, contractors and third party vendors
- All systems, applications and devices accessed using passwords
- All Novalink offices in Nigeria, Ghana and Kenya
- All remote and cloud based systems used by Novalink

---

## 3. Policy Statement

### 3.1 Password Requirements
All passwords used to access Novalink systems must meet the
following minimum requirements:

| Requirement | Standard User | Privileged User |
|---|---|---|
| Minimum Length | 12 characters | 16 characters |
| Uppercase Letters | At least 1 | At least 2 |
| Lowercase Letters | At least 1 | At least 2 |
| Numbers | At least 1 | At least 2 |
| Special Characters | At least 1 | At least 2 |
| Maximum Age | 90 days | 30 days |
| Password History | Last 10 passwords | Last 24 passwords |

### 3.2 Password Creation Rules
- Passwords must not contain the users name, username or
  date of birth
- Passwords must not contain common dictionary words
- Passwords must not contain sequential characters such as
  123456 or abcdef
- Passwords must not be reused from the last 10 previous
  passwords
- Passwords must be unique for each system and application

### 3.3 Password Management
- All employees are strongly encouraged to use the Novalink
  approved password manager to store and manage passwords
- Passwords must never be written down or stored in plain text
- Passwords must never be shared with any other person including
  IT Support staff
- IT Support will never ask for a users password
- Default passwords on all systems must be changed immediately
  upon first use

### 3.4 Multi Factor Authentication
Multi Factor Authentication (MFA) is mandatory for the following:
- All privileged and administrator accounts
- All remote access connections via VPN
- All cloud based systems and applications
- All financial systems and payment platforms
- All email accounts at manager level and above

### 3.5 Account Lockout
- User accounts will be locked after 5 consecutive failed
  login attempts
- Locked accounts must be unlocked by IT Support following
  identity verification
- Account lockout events will be logged and monitored by
  the SOC team
- Repeated lockout events will be investigated as a potential
  security incident

### 3.6 Password Reset
- Password resets must be requested through the official
  IT Support ticketing system
- Identity must be verified before any password reset is performed
- Temporary passwords issued during resets must be changed
  immediately upon first login
- Self service password reset is available for standard user
  accounts through the approved portal

### 3.7 Service and System Accounts
- Service accounts and system accounts must have passwords that
  meet privileged user requirements
- Service account passwords must be stored in the approved
  privileged access management system
- Service account passwords must be rotated every 30 days
- All service accounts must be documented and approved by
  the IT Manager

---

## 4. Roles and Responsibilities

| Role | Responsibility |
|---|---|
| CISO | Own and enforce this policy |
| IT Manager | Implement technical password controls and manage resets |
| IT Support | Process password resets following identity verification |
| GRC Manager | Monitor compliance and report violations |
| SOC Team | Monitor and investigate account lockout events |
| All Employees | Create strong passwords and comply with this policy |

---

## 5. Compliance and Enforcement
- All employees must comply with this policy at all times
- Sharing passwords is a serious violation and will result in
  immediate disciplinary action
- The IT Manager will implement technical controls to enforce
  password requirements wherever possible
- The GRC Manager will audit password compliance quarterly
  and report findings to the CISO
- Violations will be escalated in line with Novalinks
  disciplinary procedures

---

## 6. Regulatory Alignment
This policy aligns with the following regulations and frameworks:
- Nigeria Data Protection Regulation (NDPR)
- CBN Cybersecurity Framework 2022
- ISO/IEC 27001:2022 — Annex A Control 5.17
- CIS Controls v8 — Control 5
- PCI DSS v4.0 — Requirement 8

---

## 7. Review and Update Schedule
- This policy will be reviewed annually by the GRC Manager and CISO
- The policy will also be reviewed following any major security
  incident or significant organizational change
- All updates must be approved by the CISO and communicated
  to all staff
